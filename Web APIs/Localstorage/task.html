We gaan door met de laatste oefening van een Web API. In dit geval de localStorage die ook in de eindopdracht nodig zal zijn. Deze API kan data tijdelijk in je browser opslaan. Deze data zal blijven bestaan totdat iemand bewust de cache van de browser verwijderd. Oftewel, ook na een pagina refresh kun je bij deze data. Dit is erg vergelijkbaar met de werking van een Cookie, zoals je hebt geleerd bij de PHP-module.

Ik hoor je al denken; "Waarom zou ik dit gebruiken IPV een database?". Zie het als een variant die je voor zeer kleine use-cases kan gebruiken waarbij je geen back-end en/of complexe databases wilt toepassen. Let wel, het zal dus altijd gaan om dingen zoals een todo lijst. Gevoelige informatie van gebruikers zul je niet via deze methode opslaan.

Mocht je willen weten welke varianten van "lokaal opslaan van data" er zijn, kun je dit artikel lezen om te snappen hoe de varianten van localStorage precies werken. Je hoeft deze termen niet uit te hoofd te kennen, maar het is goed te weten dat er legio aan opties zijn met allemaal eigen use-cases wanneer deze te gebruiken.

Laten we aan de slag gaan met de oefening:

  • We beginnen in de formSubmitHandler(). Zorg ervoor dat je de localStorage API gebruikt om 2 waardes op te slaan. 1 waarde met key 'name' waarin de waarde uit het eerste inputveld wordt gezet. En 1 waarde met key 'age' waarin de input uit het tweede inputveld wordt gezet.
    Let goed op! We zitten weer in een submit handler van het form. Oftewel, je zult nog iets moeten doen om te voorkomen dat de pagina wordt ververst.. Kijk even terug in de les "Events > Form" als je er niet uit komt!
  • We hebben nu gezorgd dat de waardes worden opgeslagen in de localStorage. Nu willen we zorgen dat als we de pagina refreshen, dat de eerder ingevulde waardes automatisch weer worden getoond. Hiervoor gebruiken we de fillFieldsFromLocalStorage() functie.
  • De laatste stap is het verwijderen van de items uit localStorage. Zorg dat beide keys worden verwijderd en doe dit in de deleteClickHandler() functie.
    Omdat je nu weer een lege localStorage hebt zul je zien dat de fillFieldsFromLocalStorage() functie uiteraard geen waardes ophaalt voor in de HTML. Het is het meest nette een controle te schrijven of de localStorage keys wel bestaan voor je ze in de HTML input terug schrijft.
