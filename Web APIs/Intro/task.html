Nu we events onder de knie hebben kunnen we een zijstapje maken naar het onderwerp "Web API's". Dit is een verzamelnaam voor allerlei toffe features die gebruikt kunnen worden in moderne browsers (Chrome, Firefox, Edge, Safari). Deze features maken vaak gebruik van native elementen zoals geolocatie, je webcam, microfoon of batterij.

We hebben in Les 1 natuurlijk kennis gemaakt met Webservices (Pokémon, Star Wars, Instagram, Facebook, Amazon, Google, etc) waarbij deze webservices een API aanbieden die vaak via het RESTful principe werkt. Het is belangrijk te snappen dat de Web API's niks te maken hebben deze externe API's. De term API kan in veel verschillende contexten gebruikt worden, maar komt altijd neer op het gebruik van type service waar jij als ontwikkelaar gebruik van kunt maken.

Terug naar Web API's! Er zijn vele handige websites om te controleren of je een bepaalde feature of API wel kunt gebruiken in de browsers die jij wilt supporten (lees: de browsers waarvan jouw klant wilt dat de functionaliteiten werken). Twee populaire platformen die je moet kennen zijn whatwebcando.today (laat direct zien wat wel/niet werkt in de browser waarin je deze URL opent) en caniuse.com. (zoek op een JS/CSS feature en zie direct wat de support is)

Voordat je aan de slag gaat met de verschillende oefeningen is het belangrijk dat je blijft werken in je XAMPP omgeving. Veel Web API's die bepaalde toestemming nodig hebben van de gebruiker (locatie, webcam, etc.) werken alleen maar op een HTTPS URL óf via localhost (voor ons XAMPP). Hou hier dus rekening mee als je eigen projecten gaat maken en een Web API wilt testen!

Genoeg introductie gehad, tijd om aan de slag te gaan in de volgende opdracht!
