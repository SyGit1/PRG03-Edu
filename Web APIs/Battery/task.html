We gaan door met een tweede Web API, namelijk de battery. Via deze API kun je de status van de batterij van je device bijhouden. Dit kan handig zijn om gebruikers te waarschuwen of bijvoorbeeld minder features aan te roepen als de batterij in het rood komt.

Deze Web API werkt alleen in Chrome & Edge. Let hier dus op met het testen van je code. Zie de documentatie voor volledige ondersteuning per browser.

Net zoals bij de geolocation heeft het navigator object ook een optie voor de battery, in dit geval een method genaamd getBattery(). We gaan aan de slag met de volgende opdrachten:

  • We beginnen weer rustig, koppel de 2 HTML elementen aan de 2 global variables. Het is goed dit te blijven oefenen.
  • Voor de functie getBatteryStatus() is de code al gegeven omdat deze iets complexer is. Hier wordt gebruik gemaakt van 2 principes die we dit kwartaal niet zullen behandelen, namelijk Promises en Arrow functions. We raden je zeker aan beide links te lezen om in ieder geval de basis te begrijpen zodat je de code iets beter kunt begrijpen.
  • Zorg ervoor dat het element targetStatus een tekst toont waarin het percentage van de batterij getoond wordt (100%, 41%, oid). Doe dit in de functie showBatteryStatus()
    Als je een console.log() uitvoert op de battery variable kun je snel bepalen welke properties je nodig hebt om deze opdrachten goed uit te voeren.
  • Zorg ervoor dat het element targetRemaining een tekst toont die aangeeft hoeveel uur de batterij nog kan meegaan. Het bericht moet als volgt getoond worden: "Je apparaat zal uitgaan over 2,41 uur" (het getal is dynamisch uiteraard)
  • Zoals je zult merken krijg je geen getal, maar Infinity terug uit de batterij status wanneer je aangesloten bent op stroom. Zorg dat de tekst in targetRemaining in dit geval een ander bericht toont, namelijk "Je apparaat zal niet uitgaan zolang je aan de stroom zit aangesloten!"
