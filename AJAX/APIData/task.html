Nu we eenmaal data met de Fetch API hebben opgehaald van een externe API (in dit geval de Pokémon API), kunnen we verder. Laten we de data gebruiken om een mooi lijstje te maken in onze HTML met daarin de originele 151 Pokémons. Laten we werken aan de volgende opdrachten:

  • Pas je kennis toe uit "DOM > Creating" om een <ul> aan te maken in de successHandler. Geef deze UL het ID "pokemons". Maak vervolgens een <li> aan per Pokémon die in de data.results zit en plaats hierin de naam van de Pokémon. Voeg de UL toe aan de body van je HTML.
  • Kijk uit! Aangezien we data laden via een click op een button kunnen we potentieel 400 keer op de button klikken. In de getPokemonsSuccessHandler() functie willen we dus zorgen dat we niet nog een keer in deze functie komen. Verzin een slimme oplossing om te zorgen dat je maar 1 keer in de functie kunt komen.
    Er zijn veel mogelijkheden om te voorkomen dat je data dubbel in laad. Je kunt een state variable (boolean) bijhouden en daarop controleren, of deremoveEventListener() functie gebruiken. We zullen in de praktijklessen uitgebreid laten zien hoe je hier slim mee om kunt gaan in jouw code.
  • Nu we een mooi lijstje met Pokémons op ons scherm zien, willen we graag dat we na het klikken op een Pokémon de types (grass, fire, etc) zien die bij ze horen. Schrijf de code waarin bij een click op de Pokémon, zijn types in de bestaande <div> met ID "types" geplaatst worden. We hebben hiervoor de variable types al klaarstaan. Noem de callback functie van deze click pokemonClickHandler().
    Goed geraden! We moeten inderdaad een nieuwe AJAX call doen om deze extra informatie op te halen. In de data.results zit per Pokémon niet alleen de naam maar ook een nieuwe API url. Deze URL kun je weer via de Fetch API aanroepen om de extra informatie uit de API op te halen. Noem deze nieuwe callback functie getPokemonDetailSuccessHandler()
    Probeer goed te kijken hoe je zorgt dat je bij het klikken op een Pokémon daarna in de clickHandler functie nog weet wat de URL was die daarbij hoort. TIP: Het dataset attribuut kun je op een DOM Element gebruiken om informatie op te slaan die je later weer nodig hebt!
  • We zien dat we nu 2 keer de Fetch API nodig hebben. Zorg ervoor dat je maar 1 functie gebruikt om te voorkomen dat je vaker dan 1 keer dezelfde Fetch code aan het schrijven bent.
