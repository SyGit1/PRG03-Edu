Nu we het DOM hebben leren kennen, gaan we ons verdiepen in Events. In de vorige opdracht zagen we stiekem al een event voorbij komen:

let button = document.getElementById('button');
button.addEventListener('click', addItem);

Binnen deze code zien we dat een element wordt opgehaald op basis van het ID. Daarna wordt via element.addEventListener(eventType, callback); bepaald naar welk event wordt geluisterd en wat er moet gebeuren.

  • In ons voorbeeld is het element onze button.
  • In ons voorbeeld is het eventType een click op de button.
  • In ons voorbeeld is de callback de functie addItem die wordt uitgevoerd na een click op de button.

De structuur van .addEventListener wordt op elk event toegepast. Wat dus niet de bedoeling is, is het schrijven van element.onclick(callback), of in de html <button onclick="callback()">. Het is zeer belangrijk dit vanaf het eerste moment goed te doen en niet af te wijken.

In de gegeven opdracht staat ook al 1 event genoteerd, namelijk het load event op het window, die de init functie als callback uitvoert. Dit zal vanaf nu in elk voorbeeld en in elke opdracht als eerste terugkomen.

Genoeg theorie voor nu! Ga aan de slag met de volgende opdrachten op basis van de gegeven code:

  • Schrijf in de init functie de code om een 'submit' event op het formulier met ID 'new-todo-form' af te vangen. De callback is de functie addItem die al in de code staat gegeven.
  • Je drukt op de knop 'add item' en het enige wat je ziet gebeuren is dat je pagina ververst. Hoe kan dit nu gebeuren? Zoals je ziet krijg je de parameter 'e' mee. 'e' staat voor 'event' en bevat alle informatie over het zojuist uitgevoerde event (in ons geval de 'submit').

    Een krachtige optie van events die een standaardwerking hebben (forms, anchor links, etc), is dat je deze standaardwerking kunt 'blokkeren'. Zet in je addItem functie de code e.preventDefault();. Zoals je zult merken wordt het formulier niet meer verstuurd!
  • Nu mag jij hem afmaken! Gebruik de kennis uit het hoofdstuk "DOM" en maak een nieuwe <li> aan met daarin de ingevoerde tekst uit het inputveld.
  • (Bonus) Plaats in het element met ID 'message' een melding dat het item is toegevoegd aan de lijst. Maak daarbij ook het invoerveld leeg zodat er weer een nieuw item kan worden toegevoegd.
